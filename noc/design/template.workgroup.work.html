Explain tasks work flow

Again we still looking for some workflow tools to keep focus in what we need, priority what we need first and do not lost great ideas we had. :) This is my new try, read the rules in this list and lets try that tool.
  1. New tasks start in "new tasks" status. This is not a working list. Tasks here are in braisntorm mode, not approved yet. Think like a ideas list. We need talk, document, solidify idea before approve and become a working task.
  2. When task is approved, we move to "Ready to work". This list hold all tasks we need to zenofon and have to work. Leave next most need tasks at top of list.
  3. "Working this week" its a small (keep always small please) list with task we are working (and NEED finish) this week. Think like a focus tool. Better fill this list at weekend and work in the week time.
  4. "Urgent" hold tasks that do not follow normal workflow. Emergency things, and big nasty bugs comes here. This list NEED be always empty. Only fill this list with tasks that NEED interrupt work in "Working this week" list.
  5. After finish one task successfully, we move to "complete" status.
  6. All canceled task (from any status) stay in "Canceled" status for history.
As you notice, we log all changes in any task. You have to login with your id to perfect log. Go Your account (top right) and check about gravatar, we use this as avatar image.
